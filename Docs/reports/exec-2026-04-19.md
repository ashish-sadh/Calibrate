# Drift Daily Briefing — 2026-04-19

## Headline
Chat pipeline got materially smarter this period — streaming multi-item food logs, meal-period auto-detection, time-aware suggestions, and a +4.3% routing accuracy lift from a simpler intent prompt.

## Product Snapshot
Drift is an AI-first health tracker — users type what they ate or did, AI handles logging, tracking, and insights. On-device, no cloud. Currently in closed beta on TestFlight.

## What Shipped This Period
- **Multi-item food logs stream progressively** — typing "eggs, toast, and coffee" now shows each item resolving one-by-one instead of waiting for the full batch
- **Chat auto-detects meal time** — the parser assigns breakfast/lunch/dinner based on time-of-day, so "had oatmeal" at 8am logs to breakfast without asking
- **Nutrition lookup card** — "calories in chicken biryani" returns a structured macro card instead of plain text
- **Time-aware suggestion pills** — chat input shows relevant quick-actions based on time (breakfast options at 8am, workout prompts at 6pm)
- **Routing accuracy +4.3%** — simpler intent prompt lifted gold-set routing from 44/47 → 46/47; context window doubled (2048 → 4096 tokens) with long-context eval added
- **Eval coverage expanded to 225 cases** — +15 glucose/biomarkers/sleep, +23 implicit-food-intent, +25 multi-turn regression chains

## What's Working Well
- AI pipeline is now measurably better each cycle — eval-driven tuning (prompt compression 16%, context expansion, simpler intent prompt) is compounding into real user-visible improvements
- Chat feels more "alive" — streaming resolution, stage labels, and time-aware pills create perceptible responsiveness
- Auto-research optimizer + HardEvalSet found 5 real regressions this period that a human likely would have missed

## Risks & Blockers
- Food DB stuck at 2,207 — USDA API integration is the only realistic path to scale; manual entry has plateaued
- Supplement status vs logging intent is still the most common routing confusion in hard-case eval — partial fix shipped, full disambiguation still open
- No competitor has shipped on-device AI chat this good, but MFP/Whoop cloud AI keeps improving — our moat is privacy, not capability

## Focus for Next Period
1. **Per-component gold sets + multi-turn integration tests** — close the last reliability gap by testing each pipeline stage in isolation against live LLM
2. **Intent confidence calibration** — when the classifier is uncertain, should it ask vs guess? Currently guesses too eagerly on ambiguous queries
3. **Food DB: USDA chat-tier coverage** — verified nutrition for top 500 queries, routed via USDA fallback when local DB misses
